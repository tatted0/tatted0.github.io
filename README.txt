Пакет hand-coded сайта (портфолио) — содержимое:
- index.html
- styles.css
- script.js
- images/ (myphoto.jpg — замените на своё оригинальное фото; work1.jpg..work3.jpg — примеры)
- images/icon.svg (адаптивная графика)
- images/qr-placeholder.svg (QR-код-заглушка)

Инструкции по выполнению требований задания:
1) Размещение:
   - Для ручной верстки разместите файлы на доступном в РФ хостинге (например, Timeweb, Reg.ru, Yandex Cloud).
   - После загрузки получите URL вида https://your-domain.ru или https://username.timeweb.space.
   - Сгенерируйте QR-код, кодирующий финальный URL (многие хостинги/онлайн-сервисы, например qr-code-generator.ru, или командная утилита `qrencode` позволят получить PNG/SVG).
   - Замените файл images/qr-placeholder.svg на полученный QR (или укажите ссылку в отчёте).

2) Контент и уникальность:
   - Текст в разделе "Обо мне" — собственный, >300 символов (включён).
   - Замените images/myphoto.jpg на ваше оригинальное фото.
   - Адаптивная графика: images/icon.svg — готова (SVG масштабируется).

3) Адаптивность:
   - Проверьте в DevTools (эмуляция мобильных устройств) и на реальных устройствах.
   - В CSS использованы @media правила и относительные сетки.

4) Навигация:
   - Меню с 4 разделами; реализована плавная прокрутка (script.js).

5) Галерея:
   - 3 изображения в сетке; CSS Grid используется (.gallery-grid).

6) Форма обратной связи:
   - Поля: Имя, Email, Сообщение; валидация email (проверка наличия '@') реализована в script.js.
   - Для реального отправления подключите серверную часть (например, обработчик на PHP/Node.js или сервис formspree.io).

7) Футер:
   - Копирайт с годом 2025; кликабельные телефон и email в контактной секции.
   - Иконки соцсетей (Telegram, GitHub) — ссылки заполните своими реальными.

ГОСТ-отчёт:
- В отчёте о выполнении задания включите:
  1. Титульный лист (согласно ГОСТ 7.32-2017) — ФИО, тема, учебное заведение, руководитель (если требуется), дата.
  2. Содержание (оглавление).
  3. Введение: цель, задачи, инструменты (HTML/CSS/JS, хостинг).
  4. Основная часть: описание структуры сайта, список доработок, используемые технологии, адаптивные решения, доступность, тестирование (указать тесты в DevTools и на устройствах).
  5. Результаты: ссылка на сайт, вложенный QR-код (после публикации), скриншоты мобильной/десктопной версии (включите в отчёт).
  6. Заключение.
  7. Список используемой литературы/ресурсов (если есть).
  8. Приложения: исходные файлы, скрипты, заметки по валидации.

Примечание по QR: я попытался сгенерировать QR-код автоматически в среде выполнения, но если это не получилось, замените images/qr-placeholder.svg на QR, полученный после публикации сайта — это обязательный пункт задания.
